# AEGIS

# UI Specification – Report

Version: 1.0

---

# 1. Purpose

The Report page presents the final outcome of an investigation.

It transforms all collected evidence, findings and AI intelligence into a professional security assessment report suitable for technical users, developers and decision-makers.

The report should feel like a deliverable from a cybersecurity consulting firm rather than exported scanner logs.

Users should be able to read the report online or export it as PDF, HTML or Markdown without losing formatting or clarity.

---

# 2. Design Philosophy

The Report page should prioritize readability.

Think

* Stripe Documentation
* Notion
* Linear
* Apple Documentation

Avoid

* Long terminal dumps
* Dense tables
* Scanner logs
* Large red warning banners

Every page should breathe.

Typography should lead the experience.

# 6. Executive Summary

This is the first section users read.

Generated by AI using structured investigation data.

Include

Overall security posture

Key findings

Highest priority issue

Recommended next actions

Example

```
Aegis completed a Standard Investigation of example.com.

The assessment identified two High severity findings and one Medium severity finding.

No Critical vulnerabilities were detected.

The highest priority recommendation is implementing a Content Security Policy and enabling HTTP Strict Transport Security.
```

Keep the summary concise.

---

# 7. Security Score

Avoid simplistic scores like

```
92% Secure
```

Aegis should never imply absolute security.

Instead display a **Security Posture Overview**.

Example

```
Overall Posture

Moderate Risk

──────────────

Strengths

✓ Modern server

✓ HTTPS Enabled

✓ Updated PHP Version

──────────────

Areas Requiring Attention

• Missing CSP

• Missing HSTS

• Weak Security Headers
```

The emphasis is on context rather than percentages.

# 21. Success Criteria

A successful Report page should allow a user to:

* Understand the outcome of the investigation without reviewing the live scan.
* Quickly identify the most important security issues.
* Access detailed findings when needed.
* Export a professional report suitable for sharing.
* Feel confident that the report represents a structured security assessment rather than raw scanner output.

The Report page should be the polished conclusion of the Aegis investigation journey, presenting technical findings in a format that is clear, professional and actionable for both technical and non-technical audiences.
